# Whilter.AI — Behavioral Interview Stories (STAR)

Use these in behavioral rounds. Keep details NDA-safe.

## Story 1: Full-stack feature slice

**Situation:** Product needed [UI + API capability] integrated with LLM backend.  
**Task:** Own the slice from UI through API contract to integration.  
**Action:** Built React/TypeScript components, REST endpoints with validation, wired LLM calls with error handling and loading states.  
**Result:** Feature shipped to production; fewer hand-offs between frontend and backend.

## Story 2: RAG / document pipeline (IntelliRAG — high level)

**Situation:** Org needed semantic search over proprietary documents.  
**Task:** Contribute to ingestion → chunking → retrieval → LLM response flow.  
**Action:** Worked on document ingestion and API integration; ensured predictable behavior for end users.  
**Result:** Production RAG capability for internal knowledge (no client/code details).

## Story 3: Debugging under deadline

**Situation:** Integration issue blocked a release.  
**Task:** Find root cause across UI and API boundary.  
**Action:** Reproduced, traced request/response, fixed validation mismatch, added guardrails.  
**Result:** Unblocked release; documented pattern for team.

## Story 4: Learning quickly

**Situation:** New stack patterns (TypeScript, REST contracts, LLM APIs) in fast-moving startup.  
**Task:** Deliver production-quality work while ramping.  
**Action:** Paired with seniors, read existing patterns, shipped incrementally with reviews.  
**Result:** Consistent contribution within first weeks.

## Questions to ask interviewers

- How does the team split frontend/backend ownership?
- Where do LLM features sit in the roadmap?
- What does onboarding look like for the first 30 days?
